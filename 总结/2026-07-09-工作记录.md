## 二、五项改进实施详情

### ① 市场聚合架构 (★ 核心突破，架构级)

**修改文件**：`model.py`、`train.py`、`utils.py`、`config.py`

| 新增模块 | 功能 | 维度 |
|---------|------|:--:|
| `MarketAttentionPooling` | 多头注意力池化，300只股票特征 → 市场状态向量 | 64维 |
| `MarketGate` | 门控机制：市场向量调制个股排序分数 | — |
| `market_head` | 二分类器，预测未来5日市场涨跌 | 1维 |

数据流：
```
股票特征 [B,300,d_model]
  → Cross-Stock Attention
      ├→ Attention Pooling → market_embedding [B,64]
      │      └→ market_head → BCE loss (涨/跌预测)
      └→ MarketGate(stock_i, market) → 调制后排序分数
```

配置：`use_market_aggregation=True`, `market_dim=64`, `market_loss_weight=0.2`

### ② 混合标签

**修改文件**：`train.py` (`_build_label_and_clean`)

```python
rank_label = groupby('日期')['label_abs'].rank(pct=True)     # 0~1 排序质量
abs_label  = label_abs / mean(|label_abs|).clip(-2, 2)       # 保留收益幅度+方向
label = 0.7 * rank_label + 0.3 * abs_label                  # 混合
```

配置：`use_mixed_label=True`, `mixed_label_alpha=0.7`

---

### ③ Portfolio-Level 损失

**修改文件**：`train.py` (`WeightedRankingLoss`)

新增 `_portfolio_return_loss`：
- Gumbel-Softmax 松弛 Top-K 选择 → 可微
- 直接最大化选中股票的真实绝对收益期望
- 负收益施加 2x 额外惩罚
- 解决排序损失"排列等变"问题

```
total_loss = ranking_loss + 0.15 × portfolio_return_loss
```

配置：`portfolio_loss_weight=0.15`, `portfolio_temperature=0.5`

---

### ④ 市场宽度特征

**修改文件**：`utils.py`

每日截面统计量（7维）：
- `market_advance_ratio` — 涨跌家数比
- `market_return_dispersion` — 收益离散度
- `market_return_skew` — 收益偏度
- `market_volume_change` — 放量/缩量信号
- `market_up_amount_ratio` — 上涨股成交额占比
- `market_amplitude_mean` — 平均振幅
- `market_return_mean` — 市场均值
