# 2026-07-07 工作记录 — GBDT Stacking 实验

## 背景

在实验报告中讨论了替代排序算法后，决定探索 **GBDT（LightGBM LambdaRank）** 是否能在当前项目上带来提升。

## 策略设计

采用 **策略 D：Transformer 表征 + GBDT Stacking**：

1. 加载 V7 已训练好的 StockTransformer，冻结权重
2. 通过新增的 `extract_features()` 方法，提取 `ranking_layers` 输出（128 维向量）
3. 拼接原始特征最后一天的值（219 维）→ 总计 **347 维**
4. 训练 LightGBM LambdaRank，每日为一个 query group
5. 测试集预测 Top-5，计算 5 日实际收益率

## 实施过程

### 第一轮（特征维度不匹配）
- **错误：** `mat1 and mat2 shapes cannot be multiplied (17880x218 and 219x256)`
- **原因：** 错误地将 `instrument` 列从特征中排除，导致 219→218 维
- **修复：** 保留 `instrument` 列（它是模型期望的整数索引特征）

### 第二轮（LambdaRank 标签类型错误）
- **错误：** `label should be int type (met -0.003479) for ranking task`
- **原因：** LightGBM LambdaRank 要求整数相关度标签，当前是连续超额收益率
- **修复：** 新增 `continuous_to_relevance()` 函数，按天组内分位数将连续收益转为 0~9 等级，Top-5 强制标记为最高等级

### 第三轮（测试日期错误）
- **错误：** 预测的是 2026-03-06（训练集最后一天），而非测试窗口 2026-03-09~13
- **原因：** `_build_label_and_clean` 会丢弃无未来标签的测试窗口日期
- **修复：** 测试集特征工程绕过 `_build_label_and_clean`，保留测试窗口日期；收益率计算改为 day1→day5 开盘价

### 第四轮（最终运行）
- 训练集：120,290 样本，403 个 query groups
- 验证集：13,800 样本，46 个 query groups
- LightGBM 早停于第 **66** 棵树，val NDCG@5 = **0.348**
- 测试日期：2026-03-13

## 最终结果

| 方法 | 5 日收益率 |
|------|:---:|
| 官方基线 | **2.52%** |
| V6 (问题版 Transformer) | 0.46% |
| V7 Transformer (等权) | **6.02%** |
| V7 Transformer (收益门控) | **6.15%** |
| **GBDT Stacking (等权)** | **-5.32%** ❌ |
| 排行榜第 1 名 | 14.17% |

**结论：GBDT Stacking 在此任务上显著劣于纯 Transformer。**

## 失败原因分析

1. **NDCG@5 = 0.348 极低**：随机排序的 NDCG@5 约为 0.25~0.30（300 只股票中选 5 只），0.348 仅略好于随机。模型几乎没有学到有效的排序信号。

2. **特征提取断点不当**：当前从 `ranking_layers` 输出（128 维）提取表征。但该层是 Transformer 端到端训练的中间产物，其表征高度依赖后续的 `score_head` 才能转化为有效排序。GBDT 无法利用这种"协作训练"的依赖关系。

3. **离散化损失信息**：将连续超额收益转为 0~9 等级丢失了收益大小的细粒度信息，而这些信息正是 Transformer 排序损失利用的核心。

4. **忽视了时序交互**：GBDT 每个样本独立，无法利用 Cross-Stock Attention 学到的股票间相对关系。

5. **特征重要性集中在少数维度**：Top-3 特征（raw_189, raw_38, raw_37）贡献了 50%+ 的 gain，说明大量特征未被有效利用。

## 关键经验

- **GBDT Stacking 不适合此任务的当前特征提取点**。如果未来重试，应改为提取更底层的表征（如 Cross-Stock Attention 输出或 Feature Attention 输出），或在特征工程阶段就做时序汇总（策略 B）。
- **纯策略 B（时序汇总 + 无 Transformer 特征）** 可能比 Stacking 更有效——避免了 Transformer 中间表征的"不兼容"问题。
- **LambdaRank 离散化** 在这个任务上是一个实质性瓶颈。

## 后续方向

1. **策略 B（纯 GBDT + 时序汇总）** — 完全不用 Transformer 表征，直接对 60 天序列做统计汇总
2. **参数调优** — 当前只用了默认参数，可以调 num_leaves、learning_rate、lambda 等
3. **XGBoost / CatBoost 对比** — 不同 GBDT 实现在排序任务上可能有差异
4. **更底层表征提取** — 尝试从 Cross-Stock Attention 输出提取特征
