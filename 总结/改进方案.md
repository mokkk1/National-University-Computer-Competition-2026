# 沪深300指数预测 — 改进方案汇总

## 当前状态

| 指标 | 最优结果 |
|------|:--:|
| V7 自评收益率（固定窗口） | **+6.15%** |
| Enhanced Walk-Forward 平均收益 | **-1.37%** |
| 官方评测得分（V7） | -1.29% |
| 排行榜第 1 名 | +21.13% |
| 差距 | ≈22.5pp |

**核心瓶颈**：模型排序能力有效（4/6 窗口显著优于随机），但绝对收益被市场方向覆盖。竞赛只允许做多，4/6 个 Walk-Forward 窗口恰逢季末市场下跌，导致选出"跌得少"的股票仍为负收益。

---

## 一、架构级改进（★★★ 核心突破）

### 1.1 市场聚合 + 市场方向头

**问题**：Cross-Stock Attention 只做股票间对比，没有任何模块将 300 只股票的信息聚合为一个市场状态向量。模型能判断"茅台 vs 五粮液谁更好"，但不知道"白酒板块整体在加速下跌"。

**方案**：在 `StockTransformer` 中新增三个子模块：

- **全局池化层**：对 Cross-Stock Attention 输出的 300 个股票特征做均值/注意力池化 → 64 维市场向量
- **市场方向头** (`market_head`)：预测未来 5 日市场整体涨跌（二分类）
- **市场门控** (`market_gate`)：将市场向量拼回个股特征，调制排序分数

```
股票特征 [B, 300, d_model]
    → Cross-Stock Attention
        ├→ 均值池化 → market_embedding [B, 64]
        │      └→ market_head → 涨/跌 预测
        └→ concat(stock_i, market) → market_gate → 调制后分数
```

**训练**：新增 `market_loss = BCE(market_pred, market_label)`，`market_label = (当天所有股票收益均值 > 0)`。

**预期效果**：市场预测为下跌时，防御性股票（低波动、大市值）排名自动上升。

**改动量**：大（`model.py` + `train.py`），需重新训练。

---

### 1.2 TimesNet 预训练时序编码器

**问题**：当前 `temporal_encoder` 是随机初始化的 3 层 Transformer，在 21-75 天训练样本上难以学到有效的时序表示。

**方案**：用 TimesNet（ICLR 2023）替换/增强 `temporal_encoder`。TimesNet 将 1D 时序转为 2D 张量，用卷积捕获周内/月内周期模式。可使用在大规模金融数据上预训练的权重。

**改动量**：大（`model.py` temporal_encoder 替换），需引入 TimesNet 依赖。

---

## 二、标签与损失函数改进（★★ 训练策略）

### 2.1 混合标签

**问题**：`use_quantile_label=True` 将标签压缩为每日组内 [0,1] 分位数，牛市 +10% 和熊市 -10% 的标签分布完全一样。模型完全看不到绝对收益的幅度和符号。

**方案**（`train.py` `_build_label_and_clean`）：

```python
rank_label = processed.groupby('日期')['label_abs'].rank(pct=True)
abs_label  = processed['label_abs'] / (processed['label_abs'].abs().mean() + 1e-8)
alpha = 0.7
processed['label'] = alpha * rank_label + (1 - alpha) * abs_label.clip(-2, 2)
```

- α=0.7 保证排序质量仍占主导
- 1-α=0.3 保留绝对收益的符号和大致幅度
- clip(-2, 2) 防止极端值污染标签分布

**改动量**：小（一行改动），需重新训练。

---

### 2.2 Portfolio-Level 损失

**问题**：当前 `WeightedRankingLoss` 的四个子损失（Listwise/Pairwise/NDCG/Precision@K）全是排列等变的：给所有分数加同一个常数，损失不变。这导致模型不关心选出的股票实际收益是多少。

**方案**：新增 `portfolio_return_loss`，用 Soft Top-K 近似选择，直接最大化选中股票的真实绝对收益期望：

```python
def portfolio_return_loss(y_pred, y_true_abs, k=5):
    pred_prob = F.gumbel_softmax(y_pred / temperature, hard=False, dim=1)
    _, topk = torch.topk(pred_prob, k, dim=1)
    selected = y_true_abs.gather(1, topk)
    return -selected.mean() + F.relu(-selected.mean()) * 2.0
```

组合：`total_loss = ranking_loss + 0.15 * portfolio_return_loss`

**改动量**：中（loss 函数），需重新训练。

---

### 2.3 多周期标签

**问题**：当前只用 T+5 一个时间窗口的收益做标签。单一窗口噪声大，且忽略了不同持有期的收益模式。

**方案**：同时构建 T+1、T+3、T+5 三个标签：

```python
processed['label_t1'] = compute_label(horizon=1)
processed['label_t3'] = compute_label(horizon=3)
processed['label_t5'] = compute_label(horizon=5)
```

多任务学习时为每个周期各加一个排序损失，或加权融合为单一标签。

**改动量**：中（`_build_label_and_clean` + loss），需重新训练。

## 三、特征工程改进（★ 低成本增量）

### 3.1 市场宽度特征

**问题**：Enhanced 配置已引入宏观特征（国债收益率、北向资金等），但缺少市场内部结构信息——"300 只股票今天是普涨还是分化"。

**方案**：每日计算以下截面统计量：

| 特征 | 公式 | 含义 |
|------|------|------|
| `advance_ratio` | (涨跌幅 > 0 的股票数) / 300 | 涨跌家数比 |
| `return_dispersion` | Std(涨跌幅) | 收益离散度（高=分化大=选股机会） |
| `return_skew` | Skew(涨跌幅) | 正偏=多数上涨，负偏=少数暴跌拖累 |
| `volume_change` | 当日均量 / 前日均量 - 1 | 放量/缩量信号 |

作为外部特征拼接到每只股票的日特征中。

**改动量**：小（特征工程函数），需重新训练。

---

### 3.2 修复行业分类

**问题**：AKShare 行业 API 将所有 300 只成分股归为同一行业，导致 `industry_embedding` 模块完全失效。

**方案**：
- 优先：使用静态行业映射表（申万一级行业分类，约 28-31 类）
- 备选：用 `ak.stock_board_industry_cons_em()` 逐个查询
- 验证：确认至少 10+ 不同行业有分布

**改动量**：小（`macro_industry.py` 数据源替换）。

---

### 3.3 更多宏观指标

**问题**：`fetch_macro.py` 中 CPI/PMI/M2/社融等 API 因 Akshare 接口变更而获取失败，当前仅 5 个宏观指标可用。

**方案**：修复 API 调用或更换数据源，补齐：

| 指标 | 频率 | 对股市的影响 |
|------|:--:|------|
| CPI 同比 | 月 | 通胀预期 → 货币政策预期 |
| PMI | 月 | 经济景气度，领先股指 1-2 月 |
| M1/M2 同比 | 月 | 流动性宽松/收紧 |
| 社融增量 | 月 | 信用周期 |
| Shibor 隔夜 | 日 | 短期资金面 |

**改动量**：小（`fetch_macro.py` API 修复）。

## 四、后处理改进（★ 不需要重训模型）

### 4.1 市场门控后处理

**问题**：后处理阶段——无论是 `select_top_stocks_with_gate` 还是 `optimize_weights`——都没有考虑当前市场环境，牛市熊市用同一套逻辑。

**方案**：在预测阶段，根据外部市场信号动态切换选股策略：

```
if 市场信号 < 0 (预测下跌):
    选股侧重: 低波动 + 高分红 + 防御性行业
    或: 空仓（不选股）
else:
    正常收益门控选股
```

**市场信号**可选来源：
- 沪深 300 指数近 10 日涨跌（最简单，零成本）
- 宏观特征的线性组合（已有数据）
- 随机森林二分类器（低维特征，小样本友好）

**改动量**：小（`predict.py` 后处理逻辑），**不需要重训模型**。这是验证"方向判断是否有用"的最快路径。

---

### 4.2 非季末预测日期

**问题**：Walk-Forward 的 6 个预测日期全部是季末（3/31, 6/30, 9/30, 12/31），而 A 股季末常有机构调仓/窗口效应，不代表全部交易环境。

**方案**：自评时增加非季末预测日期（如每月中旬），得到更全面的性能估计。

**改动量**：小（`self_eval.py` 日期配置）。

---

## 五、替代模型探索（长期方向）

### 5.1 LSTM 替代 temporal_encoder

- **适配度**：中等。LSTM 可替换时序提取模块，但无法替代 Cross-Stock Attention
- **风险**：小样本下 LSTM 更容易过拟合
- **预期**：持平或略差，不建议优先尝试

### 5.2 市场方向分类器（RF/SVM）

- **定位**：辅助模块，不替代主模型
- **输入**：宏观特征 + 市场宽度特征（10-20 维）
- **输出**：未来 5 日市场涨/跌
- **预期**：小样本低维场景下优于 Transformer，可作为方案 4.1 的外部信号源

### 5.3 策略 B — 纯 GBDT + 时序汇总

- **定位**：基线对比，验证 Transformer 表征是否必需
- **方案**：对 60 天序列做统计汇总（均值/标准差/趋势/分位数等），送入 LightGBM/XGBoost LambdaRank
- **预期**：大概率不如 Transformer，但复杂度低，可快速验证

### 5.4 GBDT 参数调优

- 当前只用了默认参数 （`num_leaves=31`）
- 可调：`num_leaves`、`learning_rate`、`min_child_samples`、`lambda_l1`/`lambda_l2`
- 从 -5.32% 可能改善到 -2%～0%，但难以超过 Transformer

## 关键经验（来自已有实验）

1. **Walk-Forward 自评能诚实估计泛化能力**：Enhanced -1.37% ≈ V7 官方 -1.29%
2. **宏观特征是唯一被验证有效的改进**：困难窗口提升 10+pp
3. **多 seed 集成在小数据场景无效**：各 seed 学到不同噪声
4. **GBDT Stacking 不适合此任务的当前特征提取点**：中间表征与下游模型不兼容
5. **final_score ≠ 实际收益**：排序质量指标与绝对收益弱相关
6. **训练样本严重不足**：2.5M 参数在 21-75 天样本上训练，需更激进的正则化或预训练
