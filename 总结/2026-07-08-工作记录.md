# 2026-07-08 工作记录 — 模型优化与深度诊断

## 一、工作概述

在官方评测窗口关闭后，对沪深300选股模型进行了系统性优化和根因诊断。主要工作包括：
1. 实现 Walk-Forward 滚动窗口训练框架
2. 三配置对比实验（Light / Standard / Enhanced）
3. 引入宏观特征 + 行业 Embedding
4. 多 seed 集成验证
5. 定位收益率天花板的根本原因

## 二、Walk-Forward 交叉验证框架

### 2.1 动机
原 V7 模型在固定验证窗口（2026-01~03）上反复优化，自评 +6.15% 但官方得分 -1.29%，存在元过拟合。

### 2.2 实现
训练 6 个滚动窗口，每窗口独立训练-验证，最终多模型集成预测。

| 窗口 | 训练截止日 | 验证期 | 训练天数 | 样本数 |
|:---:|------|------|:---:|:---:|
| W1 | 2024-09-30 | 2024-10~11 | 181 | 21 |
| W2 | 2024-12-31 | 2025-01~02 | 242 | 32 |
| W3 | 2025-03-31 | 2025-04~05 | 299 | 42 |
| W4 | 2025-06-30 | 2025-07~08 | 359 | 51 |
| W5 | 2025-09-30 | 2025-10~11 | 420 | 64 |
| W6 | 2025-12-31 | 2026-01~03 | 483 | 75 |

### 2.3 改进点
- **分位数排序标签**（P1）：`label = rank(label_abs)` 替代超额收益，消除牛熊市偏差
- **多周期预测融合**（P2）：最近5个交易日分数取中位数
- **一致性过滤后处理**（P5）：至少 40% 模型共识才入选

## 五、根因诊断：为什么所有模型收益率都是负的？

### 5.1 数据基础

| 指标 | 值 |
|------|:--|
| 个股5日收益均值 | +0.426% |
| 个股5日收益标准差 | 5.78%（信噪比 1:13） |
| 正收益比例 | 48.9% |
| Oracle Top-5 均值 | +16.51% |
| 随机5只均值 | +0.42% |
| Top-5 相邻日重叠率 | 1-2/5（仅 20-40%） |
| Top-5 持续概率 | 0.5%（几乎随机） |

### 5.2 问题层级分析

#### 第一层：模型 vs 随机
模型在 **4/6 窗口**中显著优于随机选股：

| 日期 | 随机5只 | Enhanced | vs随机 |
|------|:--:|:--:|:--:|
| 2024-09-30 | -12.56% | -2.82% | **+3.74σ** |
| 2024-12-31 | -3.79% | -8.64% | -2.90σ |
| 2025-03-31 | -7.85% | -4.53% | **+1.18σ** |
| 2025-06-30 | +0.72% | +3.08% | **+1.59σ** |
| 2025-09-30 | -1.28% | +1.82% | **+1.13σ** |
| 2025-12-31 | +2.39% | +2.90% | +0.27σ |

**模型排序能力是有效的**，不是随机猜测。

#### 第二层：市场方向覆盖
预测日期恰好是季末（3/31, 6/30, 9/30, 12/31），**4/6 个日期后市场整体下跌**：

| 日期 | 后续市场均值 |
|------|:--:|
| 2024-09-30 | -12.58% |
| 2024-12-31 | -3.78% |
| 2025-03-31 | -7.97% |
| 2025-06-30 | +0.74% |
| 2025-09-30 | -1.25% |
| 2025-12-31 | +2.42% |

**4/6 窗口市场下跌，均值 -3.74%。**

#### 第三层：核心矛盾
竞赛规则只允许做多，模型只学排序不学方向。

```
市场下跌 -12% → 模型选出跌-3%的股票 → 收益-3%（vs 市场-12%，胜出9pp）
    ↓
但 -3% 还是负的！
```

**绝对收益为负不是排序能力问题，是市场方向问题。**

### 5.3 其他发现

- **原始特征极端冗余**：开盘/收盘/最高/最低互相 r=1.000，197 个 Alpha 特征多为共线特征的线性组合
- **训练样本严重不足**：2.5M 参数在 21-75 个样本上训练，每 epoch 仅 6-19 个 batch
- **行业 Embedding 失效**：AKShare 行业 API 将所有 300 只股票归为同一行业
- **final_score ≠ 实际收益**：排序质量指标与绝对收益弱相关

## 七、关键经验

1. **Walk-Forward 自评能诚实估计泛化能力**：Enhanced -1.37% ≈ V7 官方 -1.29%
2. **宏观特征是唯一被验证有效的改进**：困难窗口提升 10+pp
3. **多 seed 集成在小数据场景无效**：各 seed 学到不同噪声，median 聚合损失信号
4. **模型排序能力有效但被市场方向掩盖**：4/6 窗口跑赢随机，但绝对收益仍为负
5. **天花板在于缺少市场方向判断**：竞赛只允许做多 + 季末市场常回调 = 绝对收益上限

---

## 八、后续方向

1. **市场方向预测**：在模型中加入显式的市场涨跌判断，预期下跌时选择低波动/防御性持仓
2. **行业分类修复**：更换 AKShare API 或使用静态行业映射表
3. **更多宏观指标**：修复 CPI/PMI/M2 等 API 调用
4. **TimesNet 预训练模型**：替换 temporal_encoder，利用预训练时序表征
5. **多周期标签**（T+1, T+3, T+5）替代单一 T+5
6. **非季末预测日期**：Walk-Forward 窗口不应全部对齐到季末
