# ADR-3: quarter_aware 后处理而非模型内日历特征

**日期**: 2026-07-20  
**状态**: ✅ 已采纳  
**决策者**: 项目组（基于 07-10 日历特征 vs quarter_aware 门控对比实验）

---

## 背景

Walk-Forward 自评发现同一 V8 模型在不同日期的表现差异达 4.57pp：非季末日 +2.51% vs 季末日 -2.06%。这引出一个关键设计问题：如何让模型在季末环境下表现更好？

## 选项

### 选项 A: 日历特征（模型内部）
新增 `days_to_qe`（距季末天数）和 `is_qe_month`（是否季末月）两个特征，让模型在训练时学到季末的应对策略。

### 选项 B: quarter_aware 后处理（模型外部）
保持 V8 原始模型不变，在后处理阶段根据季末+市场信号的联合状态，输出显式的防御/进攻策略。

```
           │ 市场看涨    │ 市场看跌
───────────┼─────────────┼─────────────
非季末     │ 🚀 正常进攻  │ 🛡️ 防御(0.6)
季末(d<5)  │ ⚠️ 谨慎(0.5) │ 🏃 极度防御(0.95)
```

## 实验结果

| 配置 | 季末均值 | 非季末均值 | 综合均值 | 正收益 |
|------|:--:|:--:|:--:|:--:|
| V8 原始 + adaptive gate | -2.06% | +2.51% | +0.99% | 67% |
| V8 + 日历特征 + adaptive gate | -2.27% | +0.12% | -1.08% | 46% |
| **V8 原始 + quarter_aware** | **-0.37%** | **+2.51%** | **+1.55%** | **72%** |

日历特征训练 144 分钟反而退化（-1.08%），quarter_aware 零训练成本提升 1.69pp（季末）。

## 决策

**选择选项 B（quarter_aware 后处理）。**

## 理由

1. **模型知道"快到季末"但不知道"该做什么"**：日历特征是信息，不是策略。在小样本下（21-75 个训练日），模型无法从稀疏的季末日样本中学到有效的应对规则。

2. **后处理策略更高效**：零训练成本，可快速迭代规则（调节防御权重、信号阈值），不依赖模型重训。

3. **关注点分离**：模型专注排序（V8 原始），后处理专注场景适配。两者独立优化，不互相干扰。

4. **W1 改善最显著**：2024-09-30 季末日收益从 -17.97% → **-5.42%**（+12.55pp）。

## 后果

- `market_gate.py` 成为独立的后处理模块，支持 `quarter_aware`/`adaptive`/`always_normal`/`always_defensive` 四种策略
- 后续优化重点从"改模型"转向"改门控策略"
- 门控参数（防御权重、信号阈值）仍有调优空间（见 07-20 收尾总结）
- 季末+看涨场景（w=0.35）的防御力度被证明不足（W1 -10.93%）
